Case Study: AI Infrastructure

OpenCLI 🚀
Breaking the API Wall.

將複雜的 Web 介面轉化為標準化 CLI，讓 AI Agent 獲得真正的「Web 自由」。

🚩 The Challenge

現代 Web 應用程式（SPA, React, Vue）已演變為複雜的狀態機。對於 AI Agent 而言，傳統的 HTTP 請求或簡單的 Scraper 面臨三個死胡同：

  • 動態渲染：內容在客戶端生成，傳統抓取工具只能拿到空白 HTML。
  • 反爬蟲牆：Cloudflare, CAPTCHA 與複雜的 Cookie 驗證讓 Agent 頻繁失效。
  • 交互斷層：Agent 能「讀」網頁，但很難在無需 API 的情況下完成「登入 $\rightarrow$ 搜尋 $\rightarrow$ 提交」的複雜工作流。

💡 The Insight

我意識到，解決方案不在於開發更強的 Scraper，而在於「介面適配 (Interface Adaptation)」。

如果我們能將整個瀏覽器會話（Session）封裝成一個 標準的 CLI 工具，AI Agent 就不需要學習複雜的 DOM 結構，而只需要執行簡單的 Shell 指令。這將 Web 操作從「不可預測的網頁交互」轉化為「可確定性的指令執行」。

🛠️ The Execution

我構建了一套分層架構來實現這一目標：

Browser Layer $\rightarrow$ Playwright / Firecrawl (處理動態渲染與認證)
$\downarrow$
Adapter Layer $\rightarrow$ 將 DOM 狀態轉化為結構化 Markdown/JSON
$\downarrow$
CLI Interface $\rightarrow$ 提供標準化指令 (例如: opencli search "AI Agents")

這種設計讓 AI Agent 能以極低的 Token 成本，透過單一入口操作數百個不同的網站，且完全無視後端的 API 限制。

⚡ Interactive Proof: The Power of Adaptation

對比傳統的 Web 抓取與 OpenCLI 的輸出結果：

Traditional Scraper / Raw HTML
<div class="nav-wrapper">...<script src="bundle.js">...<div id="app">Loading...</div>...<footer>Copyright 2026</footer>

從「不可讀的 HTML 雜訊」到「AI 原生的結構化資訊」。
