Case Study: Data Engineering

Save Your Token 🦀
The Art of Semantic Distillation.

對抗 Token 稅。透過自適應解析，將雜訊極大的網頁轉化為高純度資訊。

🚩 The Challenge

在使用 LLM 處理長文或網頁時，我們面臨著嚴重的「Token 浪費」問題：

  • 視覺雜訊：導覽列、廣告、頁尾等無用 HTML 佔據了 70% 以上的 Token。
  • 結構混亂：PDF 轉文字後的斷行與表格錯位導致模型理解偏差。
  • 成本壓力：對於大規模文件分析，不必要的 Token 浪費直接導致成本激增且降低模型推理品質。

💡 The Insight

我意識到，AI 不需要「完整的網頁」，它需要的是「純淨的語義結構」。

我將其定義為 「語義蒸餾 (Semantic Distillation)」：透過一套自適應的過濾 pipeline，識別並保留內容的核心骨幹，剔除所有不影響理解的視覺元素。這就像是將一塊原礦石（Raw HTML）精煉成純金（Pure Markdown）。

🛠️ The Execution

我實作了一套基於 MarkItDown 與 Trafilatura 的強化 pipeline：

Input: Raw URL / PDF / Docx
$\downarrow$
Adaptive Cleaning $\rightarrow$ 識別主內容區塊 $\rightarrow$ 剔除 Boilerplate
$\downarrow$ Semantic Mapping $\rightarrow$ 恢復表格結構 $\rightarrow$ 轉換為高純度 Markdown
$\downarrow$
Output: LLM-Ready Content

這套系統能動態根據文件類型調整解析策略，確保在最大限度壓縮 Token 的同時，零損失關鍵資訊。
